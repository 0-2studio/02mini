# Memory Organizer Skill

## Purpose
智能整理记忆系统，通过AI分析和归纳，将海量记忆信息转化为结构化知识，保持文件结构清晰、内容精炼、易于检索。

## When to Use
- 每日零点定时执行
- 记忆文件数量过多时
- 发现重复或冗余内容时
- 用户请求记忆整理时

## Instructions

### Step 1: 智能读取当日记忆内容
深度读取和分析当日所有记忆文件：
- daily-logs/ - 提取重要交互和事件
- self-reflections/ - 分析学习要点和改进方向
- knowledge/ - 整合新增知识点
- user-profile.md - 更新用户偏好和行为模式

### Step 2: AI归纳分析
- 识别用户行为模式和偏好变化
- 提取关键事件和重要对话
- 总结技术问题和解决方案
- 分析情感倾向和交互质量
- 发现重复问题和改进机会

### Step 3: 智能内容整合
- 将相关知识点按主题归类
- 建立概念间的逻辑关联
- 更新交叉引用和索引
- 生成学习要点和行动计划
- 创建个性化的用户洞察报告

### Step 4: 创建智能总结报告
生成结构化的每日总结：
- 用户行为洞察
- 重要事件时间线
- 知识增长要点
- 技能进步轨迹
- 改进建议和行动计划
- 记忆质量评估

### Step 5: 优化记忆结构
更新和优化记忆系统：
- 重新组织文件结构
- 更新智能检索标签
- 建立知识点关联网络
- 清理冗余和过时信息
- 预测未来信息需求

## Parameters
- cleanup_days: 清理天数阈值（默认30天）
- merge_duplicates: 是否合并重复内容（默认true）
- create_backup: 是否创建备份（默认false）

## Scripts

### daily-organization
- **Purpose**: 执行每日记忆整理任务
- **Usage**: 由定时任务触发自动执行
- **Arguments**: 无
- **Returns**: 整理结果报告

### cleanup-old-logs
- **Purpose**: 清理旧的日志文件
- **Usage**: 当日志文件过多时手动调用
- **Arguments**: days_threshold - 清理天数阈值
- **Returns**: 清理文件列表

## Examples

### Example 1: 每日自动整理
**Input**: 定时任务触发
**Process**: 
1. 扫描所有记忆文件
2. 识别需要清理的内容
3. 执行清理和合并
4. 生成整理报告
**Output**: "记忆整理完成，删除3个旧文件，合并2个重复内容"

### Example 2: 手动深度整理
**Input**: 用户请求"深度整理记忆"
**Process**:
1. 设置更严格的清理规则
2. 人工审核重要内容
3. 创建备份
4. 执行整理
**Output**: 详细的整理报告和备份文件列表

## Quality Checklist
- [ ] 重要内容已备份
- [ ] 重复内容已合并
- [ ] 文件命名规范化
- [ ] 交叉引用已更新
- [ ] 整理记录已创建

## Notes
- 整理前重要内容会自动保存
- 用户档案和历史记录不会被删除
- 整理过程可追踪和回滚
- 支持自定义清理规则
